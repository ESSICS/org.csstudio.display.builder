Properties

Common Properties

Every widget has properties "x", "y", "width", and "height" that define its position within the display.

Every widget also has a "name" that can be used to identify a widget.

Many widgets have a "foreground_color" or "font", which define characteristics of any text or similar elements.

Specific Properties

Many widgets have some properties that are very specific to the functionality of just that widget, properties that are not found in other widgets. The Image Widget, for example, has "Data Height" and "Data Width", which describe the number of rows and columns, respectively, to show of the input data.

Property Names

In the display editor, the "Properties" panel will show each property with a name like "Text" or "Foreground Color", and a user interface for setting the value of that property. For "Text", for example, you can enter the text; and for "Foreground Color", you can select the color from a dialog.

The property names shown in the editor can be localized, i.e. translated into different languages. Each property also has an internal name that is not localized because it uniquely identifies the property in the display and also when accessed by Rules or Scripts.

The internal name is typically based on the English property name, but all lowercase with underscores instead of spaces. The internal name for "Foreground Color" is "foreground_color".

Array and Structure Properties

Some widgets contain properties that are arrays and/or structures. For example, the Multi-State LED has a "states" property that is an array. The elements of the array are in turn structures with a "value" and a "color". For access from Rules or Scripts these properties can be accessed via names (for structured properties) and indices (for array properties); for example: 

states[0].value
states[1].color
